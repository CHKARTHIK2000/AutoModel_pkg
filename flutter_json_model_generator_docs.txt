Flutter JSON Model Generator (Interactive CLI) Project Documentation

  ------------------
  PROJECT OVERVIEW
  ------------------

Flutter JSON Model Generator is a CLI developer tool that automatically
generates Dart model classes from JSON responses. Instead of directly
calling APIs, developers can simply export the API response from tools
like Postman and place the JSON file inside their Flutter project.

The generator reads the JSON file, analyzes the structure, and creates
Dart model classes automatically.

This approach avoids authentication issues, complex API requests, and
makes the tool easier to use in real development workflows.

  -------------------
  PROBLEM STATEMENT
  -------------------

When integrating APIs in Flutter applications, developers must manually
create Dart model classes from JSON responses.

Typical workflow today:

1.  Call API using Postman
2.  Copy JSON response
3.  Manually write Dart model
4.  Implement fromJson()
5.  Implement toJson()
6.  Handle nested objects
7.  Handle optional fields

This process is repetitive and error‑prone.

  ----------
  SOLUTION
  ----------

Provide a CLI tool that:

1.  Reads a JSON file containing an API response
2.  Detects JSON structure
3.  Determines field types automatically
4.  Generates Dart model classes
5.  Saves the models in the Flutter project

Developers only need to export the JSON response once.

  --------------------
  DEVELOPER WORKFLOW
  --------------------

Step 1 – Test API in Postman

Developer tests the API and receives JSON response.

Example:

{ “id”: 1, “name”: “John”, “email”: “john@gmail.com” }

Step 2 – Save Response

Developer saves the JSON response as:

response.json

inside the Flutter project root.

Example:

project_root/ response.json

Step 3 – Run Generator

dart run api_model_generator

Step 4 – Interactive CLI

The tool asks questions:

Enter model name: User Enter output folder: lib/models Is response a
list? (y/n)

Step 5 – Model Generated

The tool creates:

lib/models/user.dart

Example output structure:

lib/ models/ user.dart address.dart

  -------------------
  ADVANCED FEATURES
  -------------------

Auto Model Update

When API response changes, developer updates the JSON file and runs the
generator again.

The tool compares:

Existing model fields vs JSON fields

New fields are added automatically.

Example:

Old model fields: id name

New JSON fields: id name phone

The tool updates the model with the new field.
